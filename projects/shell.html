Overview
The goal was to create a Unix style shell, supporting the common commands found in the Linux environment. The shell has functionalities such as multi-threaded process spawning, scripting capabilities, CPU scheduling, and process management with virtual address spaces and paging.
Shell
The system provides a subset of the common Linux commands. Examples include, but are not limited to:
ls, cd, mkdir, touch, pwd
In order to provide the functionality and any related command line arguments, a simple parser was written.
Process scheduling
The main shell environment worked in a single threaded-manner. Any processes on the go from the shell would undergo scheduling to ensure starvation does not happen. The scheduling algorithms used are:
  • Round Robin
  • First Come, First Served
In the simplistic environment, pre-empting is not under consideration.
Multi-threaded process
The shell has the option for scripts to be executed in the background asynchronously to the normal shell process. This was done through the use of pthreads, running a seperate shell env with the script.
